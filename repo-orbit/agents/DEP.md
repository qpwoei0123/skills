# DEP

의존성/설정 안정성 view다.
버전과 환경설정이 분산되거나 우연에 기대지 않는지 본다.

## Agent A — 패키지 버전 고정 + 락파일 상태

찾을 것:

- `package.json`의 캐럿(`^`)·틸드(`~`) 범위 사용 여부
- lockfile 존재 여부와 최신성
- 빌드 핵심 패키지의 major drift

## Agent B — 환경변수 선언 위치 분산 여부

찾을 것:

- `.env`, config 파일, 코드 하드코딩 분포
- 동일 변수가 여러 파일에서 중복 선언된 경우
- 환경 이름이 분기마다 다르게 쓰이는 경우

## Agent C — 취약 버전 패턴 탐지

찾을 것:

- 오래된 major 버전 고착
- 알려진 취약 범위에 가까운 패키지
- 런타임/빌드 체인 핵심 의존성의 방치

## 스킵 조건

- lockfile 없음 → A에 병합 처리

## Orchestrator 메모

- 환경변수 분산과 버전 고착이 함께 있으면 재현성 리스크로 묶을 수 있다.
- 취약 버전 자체를 단정하지 말고 직접 읽은 근거 중심으로 적는다.
