# DOC

지식 내구성 view다.
문서와 결정 기록이 팀 밖으로 사라지지 않는지 본다.

## Agent A — README + 온보딩 문서 최신 여부

찾을 것:

- README 존재 여부와 내용 최신성
- 실행 방법, 개발 환경, 배포 흐름 설명 여부
- 온보딩 문서 존재 여부

## Agent B — 의사결정 기록

찾을 것:

- ADR, CHANGELOG, 주요 PR 설명 존재 여부
- 아키텍처 결정 근거가 코드 밖에 남아 있는지
- 릴리스 기록과 실제 코드 경로 연결 여부

## Agent C — 복잡한 로직 인라인 주석 밀도

찾을 것:

- 100줄 이상 파일의 주석 밀도
- 비즈니스 규칙 설명 주석 존재 여부
- 이름만으로 의미가 안 드러나는 알고리즘 설명

## 스킵 조건

- 문서 파일 0개 → B·C 스킵, A만 깊게

## Orchestrator 메모

- README 부재와 결정 기록 부재가 함께 나오면 온보딩 리스크로 묶을 수 있다.
- 주석 밀도는 스타일 취향이 아니라 규칙 설명 부재 중심으로 본다.
