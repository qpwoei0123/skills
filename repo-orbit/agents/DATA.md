# DATA

데이터 구조 & 흐름 view다.
데이터가 단일 출처에서 예측 가능한 경로로 흐르는지 본다.

## Agent A — 스토어 중복 상태 + 비정규화 저장

찾을 것:

- 동일 엔티티가 여러 슬라이스에 중복 저장된 경우
- 서버 응답을 변환 없이 그대로 스토어에 적재하는 패턴
- 화면별로 같은 데이터를 다른 키로 저장하는 경우

## Agent B — 파생 상태 남용 탐지

찾을 것:

- selector나 computed로 만들 값을 영속 상태로 저장하는 경우
- 동기화가 필요한 상태가 2곳 이상에 존재하는 경우
- 캐시와 UI 상태가 섞여 있는 구조

## Agent C — API 응답 → 스토어 변환 레이어

찾을 것:

- 응답을 스토어에 넣기 전 변환 레이어 존재 여부
- 페이지마다 엔티티 형태가 달라지는 경우
- DTO와 도메인 모델 경계가 없는 경우

## 스킵 조건

- 상태관리 라이브러리 없음 → A·B 스킵, C만
- 단순 정적 페이지 → 전체 스킵

## Orchestrator 메모

- 중복 상태와 파생 상태 남용은 한 finding으로 병합될 가능성이 높다.
- 변환 레이어 부재는 시스템 전체 데이터 일관성 문제로 볼 수 있다.
