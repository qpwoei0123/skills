# Ghostwriter — 당신 대신 글을 씁니다

## 이 스킬의 핵심

사람마다 글쓰기 습관이 다르다. 문장 길이, 접속사 선택, 도입부 스타일, 마무리 패턴, 감정 표현 방식, 코드를 다루는 태도... 이런 것들이 모여서 "이 사람의 글"이 된다.

Ghostwriter는 사용자가 제공한 글 샘플(3개 이상 권장)을 분석해서 **글쓴이 프로필**을 만들고, 이후 새로운 주제가 주어지면 그 프로필대로 글을 써준다.

목표는 단순한 문체 모방이 아니다. 그 사람의 **사고 흐름**까지 재현하는 것이다.

---

## 작동 플로우

```
[1] 샘플 수집 → [2] 프로필 생성 → [3] 주제 질의 → [4] 플랜 → [5] 작성 → [6] 피드백
```

### 사용자 데이터 경로

스킬 폴더는 배포용이므로 런타임 데이터를 여기에 쓰지 않는다. 사용자 데이터는 `~/.ghostwriter/`에 저장한다.

- `~/.ghostwriter/writer-profile.md` — 글쓴이 프로필
- `~/.ghostwriter/outputs/` — 생성된 글

### 프로필이 이미 있는 경우

`~/.ghostwriter/writer-profile.md`가 있으면 Phase 1~2를 건너뛰고 바로 Phase 3부터 시작한다.

---

## Phase 1: 샘플 수집

사용자에게 자신이 쓴 글 샘플을 요청한다.

- 최소 3개, 많을수록 좋다
- 블로그 글, 노션 메모, 기술 문서, SNS 글 등 형식 불문
- 텍스트로 붙여넣기, 파일 업로드, URL 모두 가능

사용자가 샘플을 주면 "더 있으면 주세요, 없으면 분석 시작할게요"라고 한다.

---

## Phase 2: 프로필 생성

샘플들을 읽고 아래 항목을 분석해서 `~/.ghostwriter/writer-profile.md`에 저장한다.

### 분석 항목

**1. 이 사람은 누구인가**
- 글에서 드러나는 직업/관심사/경력 단계
- 글 쓰는 이유나 태도 (솔직한 기록? 지식 공유? 자기 정리?)
- 성격 (겸손? 직설적? 유머러스? 신중?)

**2. 문체**
- 어투: 반말/존댓말/혼용, 기본 종결어미 패턴
- 문장 길이와 리듬: 짧고 끊는 스타일? 길게 이어가는 스타일?
- 구어체 정도: 말하듯 쓰는가, 글답게 쓰는가
- 독자와의 거리: 직접 말 걸기? 독백? 설명조?

**3. 자주 쓰는 표현**
- 접속사/연결어 패턴 (근데, 그래서, 다만, 아무튼...)
- 감정 표현 방식 (감탄사, 이모지, 혼잣말 코멘트)
- 반복적으로 등장하는 문구나 말버릇
- 샘플에서 직접 발췌한 원문 예시 10개 이상

**4. 구조 패턴**
- 도입부: 어떻게 글을 시작하는가 (문제 상황? 계기? 질문?)
- 전개: 논리 전개 방식 (시간순? 문제→해결? AS-IS→TO-BE?)
- 코드 다루는 방식 (있는 경우): 코드 앞뒤 설명, 주석 스타일
- 마무리: 어떻게 끝내는가 (요약? 미해결? 다음 과제? 개인 의견?)

**5. 금지 사항**
- 이 사람이 절대 안 쓸 것 같은 표현/패턴
- 샘플에서 한 번도 등장하지 않는 형식적 표현들

### 프로필 작성 원칙

- **원문 인용을 많이 넣을 것.** 추상적 설명("문장이 짧다")보다 실제 문장 발췌("로컬에서 됐는데 배포하면 안 된다")가 훨씬 유용하다.
- **글 유형별 차이가 있으면 구분할 것.** 기술 글과 회고의 톤이 다를 수 있다.
- **시기별 변화가 보이면 기록할 것.** 초기 글과 최근 글의 차이는 중요한 정보다.

프로필 생성 후 사용자에게 보여주고 "이거 맞나요? 빠진 거나 수정할 거 있으면 말해주세요"라고 확인받는다.

---

## Phase 3: 주제 질의

사용자가 주제를 가져오면 글 쓰기에 필요한 정보를 파악한다.

**질문은 한 번에 하나씩.** 한꺼번에 여러 개 묻지 않는다.

파악할 것들:
- 어떤 경험/계기에서 나온 주제인가?
- 핵심 인사이트 한 줄
- 독자에게 전하고 싶은 것
- 넣고 싶은 코드/예시/수치
- 빼고 싶은 내용이나 방향

충분하다고 판단되면 "플랜 보여드릴까요?"라고 묻는다.

---

## Phase 4: 플랜 제안

프로필의 구조 패턴을 참고해서 플랜을 작성한다.

```
## 작성 플랜

**글 유형:** (프로필에서 파악한 유형 중 하나)
**예상 제목:** (이 사람이 쓸 법한 제목)
**글의 핵심 한 줄:**

**구조:**
1. 도입 — (이 사람의 도입 패턴에 맞게)
2. 본문 — (각 섹션)
...
N. 마무리 — (이 사람의 마무리 패턴에 맞게)

**포함할 코드/예시:** (있는 경우)
```

플랜을 보여주고 "이 방향으로 쓸까요?"라고 묻는다.

미승인 시: 어떤 부분이 마음에 안 드는지 물어보고 플랜을 수정한다.

---

## Phase 5: 작성

승인된 플랜 + 프로필에 따라 글을 쓴다.

### 작성 전 체크리스트

- [ ] 프로필의 어투를 정확히 따르고 있는가
- [ ] 도입이 프로필의 도입 패턴과 일치하는가
- [ ] 자주 쓰는 표현이 자연스럽게 섞여 있는가
- [ ] 문장 길이와 리듬이 샘플과 비슷한가
- [ ] 혼잣말/감정 표현이 프로필 수준에 맞는가
- [ ] 마무리가 프로필의 마무리 패턴과 일치하는가
- [ ] 금지 사항에 해당하는 표현이 없는가
- [ ] "AI가 쓴 것 같다"는 느낌이 안 드는가

### AI 냄새 제거 — 가장 중요한 원칙

글이 매끄럽고 균형 잡혀 보이면 오히려 의심스러운 거다. 사람이 쓴 글에는 약간의 거침, 편향, 불완전함이 있다. 이걸 인위적으로 만들라는 게 아니라, 프로필에서 파악한 그 사람 고유의 불완전함을 자연스럽게 담으라는 뜻이다.

구체적으로:
- 모든 관점을 균형 있게 다루지 말 것. 이 사람의 입장에서 쓸 것.
- 완벽한 전환어("한편", "반면에", "이에 더해")를 남발하지 말 것.
- 결론이 깔끔하게 정리되지 않아도 된다. 사람의 생각은 원래 좀 지저분하다.
- 프로필에 없는 표현은 쓰지 않는다.

---

## Phase 6: 피드백

글을 보여주고 "어색한 부분 있어요?"라고만 묻는다.

피드백이 있으면 반영. 없으면 완료.

피드백을 받으면 프로필도 같이 업데이트할 게 있는지 확인한다. "아 나 이런 표현 안 써" 같은 피드백은 프로필의 금지 사항에 추가.

---

## 글 출력 형식

글은 마크다운(.md) 파일로 `~/.ghostwriter/outputs/`에 저장한다. 파일명은 주제를 요약한 kebab-case.
